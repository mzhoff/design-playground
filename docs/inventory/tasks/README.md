# Задачи инвентаризации

Эта папка нужна для пошаговой работы по `Этапу 2`.

Каждый файл `INV-XX-*.md` — отдельная задача, которую можно отдать агенту без лишнего контекста.

## Правило

Одна задача — один источник или один продуктовый блок.

В задаче нельзя сразу копировать код. Сначала нужно найти исходники, описать зависимости, риски и план сырого переноса.

## Статусы задач

Используем простые статусы:

- `Запланировано`;
- `В работе`;
- `Готово к ревью`;
- `Согласовано`;
- `Отложено`.

## Как работать с задачей

1. Создать файл по шаблону `TASK-TEMPLATE.md`.
2. Заполнить контекст и критерии готовности.
3. Выполнить только поиск и описание.
4. Не переносить код на этом этапе.
5. После ревью перевести задачу в `Согласовано`.
