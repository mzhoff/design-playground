# apps/storybook

Техническая документация компонентов: props, states, variants, интерактивные сценарии, accessibility и визуальные проверки.

Иерархия Storybook проектируется отдельно в RFC и должна отражать не только атомы, но и вертикальные наборы: Chat Assistant, Canvas, Admin CRUD, Forms, Text/WYSIWYG, Website Blocks.
