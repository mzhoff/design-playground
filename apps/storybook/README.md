# apps/storybook

Техническая документация компонентов: props, состояния, варианты, интерактивные сценарии, доступность и визуальные проверки.

## Роль Storybook

Storybook нужен не как единственная витрина дизайн-системы, а как технический стенд качества.

Он должен помогать разработчику быстро понять:

- как импортировать компонент;
- какие props поддерживаются;
- какие есть состояния;
- какие варианты разрешены;
- как компонент ведет себя при взаимодействии;
- какие ограничения по доступности и UX нужно соблюдать.

## Планируемая иерархия

Иерархия Storybook должна отражать не только атомарные элементы, но и вертикальные наборы интерфейса:

- примитивы;
- формы;
- навигация;
- тексты и редакторы;
- чат-ассистент;
- канвас;
- админские CRUD-интерфейсы;
- графики и аналитика;
- блоки сайта;
- анимационные паттерны.
