# apps/docs

Приложение для понятной документации дизайн-системы: для команды, клиентов и будущей внешней упаковки продукта.

На первом этапе можно использовать готовый фреймворк документации, например Nextra. Собственный движок документации не строим, пока не появится понятная продуктовая или техническая причина.

## Роль приложения

`apps/docs` отвечает за объяснение смысла компонентов и паттернов:

- зачем компонент существует;
- когда его использовать;
- когда его не использовать;
- какие UX-правила у него есть;
- какие есть примеры и ограничения;
- как это объяснять команде и клиентам.

Технические props, состояния и интерактивные проверки живут в Storybook. Документация здесь должна быть ближе к продуктовой и методологической упаковке.
